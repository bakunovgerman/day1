# Исчерпывающая документация API для управления облачными виртуальными машинами (CloudVM Management API v2.0)

**Версия документации:** 2.0.1  
**Последнее обновление:** 15 октября 2023 г.  
**Базовый URL:** `https://api.cloudvm.com/v2`

## 1. Введение и аутентификация

### 1.1 Обзор API

CloudVM Management API представляет собой RESTful веб-сервис, предоставляющий программируемый интерфейс для полного управления жизненным циклом виртуальных машин в облачной инфраструктуре. API позволяет создавать, изменять, отслеживать и удалять виртуальные машины, управлять их конфигурацией, сетями, хранилищем и безопасностью.

API построен на архитектуре REST, использует HTTP-методы (GET, POST, PUT, DELETE, PATCH) и возвращает данные в формате JSON. Все запросы должны выполняться по протоколу HTTPS для обеспечения безопасности передаваемых данных.

### 1.2 Ключевые возможности

- **Полный жизненный цикл ВМ**: Создание, запуск, остановка, перезагрузка, удаление
- **Управление ресурсами**: Изменение CPU, памяти, дискового пространства
- **Сетевая конфигурация**: Управление IP-адресами, firewall правилами, балансировщиками нагрузки
- **Управление хранилищем**: Подключение/отключение дисков, создание snapshot'ов
- **Мониторинг и метрики**: Получение статистики использования ресурсов
- **Резервное копирование**: Настройка и управление политиками backup
- **Масштабирование**: Вертикальное и горизонтальное масштабирование
- **Шаблоны и образы**: Работа с предварительно настроенными образами ОС
- **Теги и организация**: Категоризация ресурсов для удобства управления
- **Управление доступом**: Ролевая модель доступа (RBAC)

### 1.3 Терминология

- **ВМ (Virtual Machine)**: Виртуальная машина - инстанс вычислительной среды
- **Инстанс**: Экземпляр виртуальной машины
- **Образ (Image)**: Шаблон операционной системы и предустановленного ПО
- **Snapshot**: Моментальный снимок состояния ВМ или диска
- **Flavor**: Конфигурационный шаблон (CPU, RAM, диск)
- **Availability Zone**: Зона доступности - логическая группа ресурсов
- **Region**: Географический регион расположения дата-центра
- **SSH Key**: Ключ для безопасного доступа к ВМ
- **Floating IP**: Публичный статический IP-адрес
- **Security Group**: Группа правил firewall

### 1.4 Регионы и зоны доступности

API поддерживает мультирегиональную архитектуру. Каждый регион представляет собой географически изолированный дата-центр с независимой инфраструктурой.

**Доступные регионы:**
- `us-east` - Восточное побережье США (Вирджиния)
- `us-west` - Западное побережье США (Орегон)
- `eu-central` - Европа (Франкфурт)
- `eu-west` - Европа (Ирландия)
- `asia-east` - Азия (Тайвань)
- `asia-south` - Азия (Мумбаи)
- `au-southeast` - Австралия (Сидней)

В каждом регионе доступно несколько зон доступности (обычно 3), обозначаемых как `zone-a`, `zone-b`, `zone-c`.

### 1.5 Аутентификация

#### 1.5.1 Схемы аутентификации

API поддерживает три схемы аутентификации:

1. **API Key Authentication** (Основная рекомендуемая схема)
2. **OAuth 2.0** (Для интеграций сторонних приложений)
3. **JWT Tokens** (Для микросервисной архитектуры)

#### 1.5.2 Аутентификация с помощью API Key

Каждый запрос к API должен содержать заголовок `X-API-Key` с вашим секретным ключом. Ключ генерируется в панели управления CloudVM.

**Пример заголовка:**
```http
X-API-Key: <YOUR_API_KEY_HERE>
```

**Структура API Key:**
- Префикс указывает тип ключа:
  - `sk_test_` - тестовый ключ (только для тестовой среды)
  - `sk_live_` - рабочий ключ (для production среды)
- 40 символов hex-строки (0-9, a-f)

#### 1.5.3 Запрос API Key

Для получения API Key выполните следующие шаги:

1. Войдите в панель управления CloudVM
2. Перейдите в раздел "API Keys" в настройках аккаунта
3. Нажмите "Generate New Key"
4. Скопируйте ключ (он покажется только один раз!)
5. Сохраните ключ в безопасном месте

#### 1.5.4 Безопасное хранение ключей

**Рекомендации по безопасности:**
- Никогда не коммитьте API ключи в системы контроля версий
- Используйте переменные окружения или секретные хранилища
- Регулярно ротируйте ключи (каждые 90 дней)
- Создавайте отдельные ключи для разных окружений
- Используйте наименьшие необходимые привилегии

**Пример настройки переменных окружения:**
```bash
# Linux/macOS
export CLOUDVM_API_KEY="sk_live_abcdef1234567890"
export CLOUDVM_API_REGION="us-east"

# Windows PowerShell
$env:CLOUDVM_API_KEY="sk_live_abcdef1234567890"
$env:CLOUDVM_API_REGION="us-east"
```

#### 1.5.5 OAuth 2.0 Аутентификация

Для сторонних приложений рекомендуется использовать OAuth 2.0. Зарегистрируйте приложение в панели управления CloudVM для получения `client_id` и `client_secret`.

**Поток авторизации:**
1. Перенаправление пользователя на страницу авторизации CloudVM
2. Пользователь подтверждает доступ
3. Перенаправление обратно с кодом авторизации
4. Обмен кода на access token

**Пример запроса access token:**
```http
POST /oauth/token HTTP/1.1
Host: auth.cloudvm.com
Content-Type: application/x-www-form-urlencoded

grant_type=authorization_code&
code=AUTHORIZATION_CODE&
client_id=CLIENT_ID&
client_secret=CLIENT_SECRET&
redirect_uri=REDIRECT_URI
```

#### 1.5.6 JWT Аутентификация

Для микросервисных архитектур поддерживается JWT. Токен должен передаваться в заголовке `Authorization`.

**Формат:**
```http
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

**Генерация JWT:**
```javascript
// Пример генерации JWT
const jwt = require('jsonwebtoken');
const token = jwt.sign(
  {
    sub: 'user_id',
    iat: Math.floor(Date.now() / 1000),
    exp: Math.floor(Date.now() / 1000) + (60 * 60) // 1 час
  },
  'your-secret-key',
  { algorithm: 'HS256' }
);
```

#### 1.5.7 Подпись запросов (Request Signing)

Для повышенной безопасности критически важных операций рекомендуется подписывать запросы.

**Алгоритм подписи:**
1. Соберите строку для подписи:
   - Метод HTTP (GET, POST, etc.)
   - Путь запроса
   - Таймстамп (Unix timestamp в секундах)
   - Nonce (уникальная строка)
   - Тело запроса (хеш SHA256)

2. Подпишите строку с помощью приватного ключа:
   ```
   signature = HMAC-SHA256(api_secret, string_to_sign)
   ```

3. Добавьте заголовки:
   ```http
   X-Timestamp: 1609459200
   X-Nonce: 7e2b3d4f5a6b7c8d
   X-Signature: 1a2b3c4d5e6f...
   ```

#### 1.5.8 Истечение запросов

Все подписанные запросы должны быть обработаны в течение 5 минут от времени указанного в `X-Timestamp`. В противном случае запрос будет отклонен.

#### 1.5.9 Тестовая и рабочая среды

API предоставляет две среды:

1. **Тестовая среда**: `https://api-sandbox.cloudvm.com/v2`
   - Бесплатное использование с ограничениями
   - Тестовые данные не сохраняются
   - Идеально для разработки и тестирования

2. **Рабочая среда**: `https://api.cloudvm.com/v2`
   - Реальные ресурсы и биллинг
   - Гарантии SLA
   - Полный функционал

#### 1.5.10 Проверка аутентификации

Для проверки действительности API ключа используйте эндпоинт:

```http
GET /auth/verify
```

**Пример успешного ответа:**
```json
{

  "account_id": "acc_1234567890",
```

#### 1.5.11 Отзыв ключей

В случае компрометации ключа немедленно отзовите его:

```http
POST /auth/keys/{key_id}/revoke
```

Или через панель управления:
1. Перейдите в "API Keys"
2. Найдите скомпрометированный ключ
3. Нажмите "Revoke"
4. Сгенерируйте новый ключ

#### 1.5.12 Ограничения по IP

Для повышенной безопасности вы можете настроить whitelist IP-адресов с которых разрешены запросы:

```http
PUT /auth/ip-restrictions
{
  "enabled": true,
  "ip_ranges": ["203.0.113.0/24", "2001:db8::/32"],
  "bypass_ips": ["198.51.100.1"]
}
```

### 1.6 Версионирование API

API использует семантическое версионирование в URL пути. Текущая версия v2. При внесении обратно несовместимых изменений будет выпущена новая мажорная версия.

**Поддерживаемые версии:**
- v1 (устарела, поддержка до 2024-12-31)
- v2 (текущая, рекомендованная)
- v3 (в разработке)

**Политика депривации:**
- Минорные версии добавляют функциональность без изменения существующих эндпоинтов
- Устаревшие функции отмечаются в документации за 6 месяцев до удаления
- Критические обновления безопасности требуют немедленного обновления

#### 2.1.1 Получение списка виртуальных машин

| `status` | string | Нет | Фильтр по статусу | `running` |
| `tag` | string | Нет | Фильтр по тегу | `production` |
| `flavor_id` | string | Нет | Фильтр по типу инстанса | `cpu-4-ram-8` |
| `search` | string | Нет | Поиск по имени или ID | `web-server` |
| `page` | integer | Нет | Номер страницы (по умолч. 1) | `2` |
| `per_page` | integer | Нет | Количество на странице (1-100) | `50` |
| `sort_by` | string | Нет | Поле для сортировки | `created_at` |
| `sort_order` | string | Нет | Порядок сортировки (`asc`/`desc`) | `desc` |
| `created_after` | timestamp | Нет | Фильтр по дате создания | `2023-01-01T00:00:00Z` |
| `created_before` | timestamp | Нет | Фильтр по дате создания | `2023-12-31T23:59:59Z` |

**Доступные статусы:** `creating`, `pending`, `running`, `stopped`, `stopping`, `starting`, `rebooting`, `deleting`, `error`, `migrating`, `resizing`, `snapshotting`

#### 2.1.2 Создание виртуальной машины

**Эндпоинт:** `POST /vms`

**Описание:** Создает новую виртуальную машину с указанными параметрами.

**Параметры тела запроса:**
| Параметр | Тип | Обязательный | Описание | Ограничения |
|----------|-----|--------------|----------|-------------|
| `name` | string | Да | Имя виртуальной машины | 2-64 символа, буквы, цифры, дефисы |
| `flavor_id` | string | Да | Идентификатор конфигурации | Должен существовать в регионе |
| `image_id` | string | Да | Идентификатор образа ОС | Должен быть совместим с flavor |
| `region` | string | Да | Регион размещения | Один из доступных регионов |
| `zone` | string | Нет | Зона доступности | Если не указана, выбирается автоматически |
| `network_id` | string | Нет | Идентификатор сети | Если не указана, используется default |
| `ssh_key_id` | string | Нет | Идентификатор SSH ключа | Обязателен для Linux образов |
| `user_data` | string | Нет | Cloud-init конфигурация | Base64 encoded, макс. 64KB |
| `tags` | array[string] | Нет | Теги для классификации | Макс. 10 тегов по 32 символа |
| `metadata` | object | Нет | Пользовательские метаданные | Ключ-значение, макс. 50 пар |
| `security_groups` | array[string] | Нет | Группы безопасности | Массив ID групп безопасности |
| `volume_size_gb` | integer | Нет | Размер корневого диска | Минимум зависит от образа |
| `assign_public_ip` | boolean | Нет | Назначить публичный IP | По умолчанию false |
| `backup_policy_id` | string | Нет | Политика резервного копирования | Если не указана, backup отключен |
| `enable_monitoring` | boolean | Нет | Включить мониторинг | По умолчанию true |
| `script_id` | string | Нет | ID скрипта для запуска при старте | Должен существовать |

Рейн встал, потянулся, и его нейроимплант мигнул уведомлением: "Срочный вызов от Адмирала Ксавьера". Он активировал голоэкран, и перед ним материализовался суровый силуэт адмирала. "Тарлок, у нас проблема. Исчезновение грузовика с квантовыми ядрами на маршруте Альфа-9. Пиратская атака или хуже — саботаж Корпорации Омега. Возьмёшь 'Нова-7' и разберёшься. Время — критично". Рейн кивнул, уже надевая свой боекомплект из нано-ткани, адаптирующейся под любую угрозу.

ETA — 14 часов в варп-пространстве", — пропела Эхо мелодичным голосом. Рейн уселся в капитанское кресло, и корабль рванул в гиперпрыжок, оставляя за собой искры искажения реальности.

Альфа-9 оказалась астероидным полем, усеянным обломками. Грузовик дрейфовал, разорванный пополам, но без следов боя. "Сканеры показывают остаточное излучение нейтронного взрыва. Это не пираты", — доложила Эхо. Рейн надел скафандр и вышел в открытый космос. Среди обломков он нашёл чёрный ящик — устройство Корпорации Омега с логотипом в виде спирали. "Саботаж. Они монополизируют квантовые ядра, чтобы поднять цены на энергию по всей Империи".

Вернувшись на борт, Рейн связался с базой: "Адмирал, это Омега. У них есть агент внутри Флота". Но связь прервалась помехами. Эхо вспыхнула тревогой: "Обнаружены три корвета класса 'Васп'. Они нас блокировали". Бой начался мгновенно. Плазменные залпы осветили вакуум, 'Нова-7' маневрировала между астероидами, уклоняясь от торпед. Рейн лично управлял турелью, сбивая врагов один за другим. Последний корвет взорвался в ослепительной вспышке.

Но победа была пирровой. Гипердвигатель повреждён, и 'Нова-7' дрейфовала к ближайшей планете — Тenebris-4, миру-улью с 20 миллиардами жителей, где правят кланы контрабандистов. Приземлившись в доках Нижнего Уровня, Рейн встретил местного информатора — эльфийку по имени Лира, с глазами цвета изумрудов и татуировками нейронных цепей. "Ты ищешь Омегу? Они здесь, в Подземелье. Но цена — 5000 кредитов и услуга", — шепнула она.

Рейн согласился. Лира провела его через лабиринт туннелей, где гудели фабрики дронов и торговали органами. В сердце Подземелья они нашли лабораторию: ряды криокапсул с квантовыми ядрами, охраняемые киборгами. Битва вспыхнула снова — лазеры, виброклинки, хакерские атаки на импланты. Лира отключила щиты, Рейн забрал ядра и поджёг лабораторию. Взрыв сотряс улей.

Улетая с Тenebris-4, Рейн и Лира стали партнёрами. "Куда дальше?" — спросила она. "В сердце Империи. Омега заплатит за всё". Их путь лежал через Пустоши Варпа, где обитали древние ужасы — тени забытых цивилизаций. Но это была лишь начало...

"Тарлок, у нас проблема. Исчезновение грузовика с квантовыми ядрами на маршруте Альфа-9. Пиратская атака или хуже — саботаж Корпорации Омега. Возьмёшь 'Нова-7' и разберёшься. Время — критично". Рейн кивнул, уже надевая свой боекомплект из нано-ткани, адаптирующейся под любую угрозу.

'Нова-7' была его гордостью: корвет с гипердвигателем класса "Феникс", вооружённый плазменными торпедами и ИИ-пилотом по имени Эхо. "Капитан, курс на Альфа-9 заложен. ETA — 14 часов в варп-пространстве", — пропела Эхо мелодичным голосом. Рейн уселся в капитанское кресло, и корабль рванул в гиперпрыжок, оставляя за собой искры искажения реальности.

Альфа-9 оказалась астероидным полем, усеянным обломками. Грузовик дрейфовал, разорванный пополам, но без следов боя. "Сканеры показывают остаточное излучение нейтронного взрыва. Это не пираты", — доложила Эхо. Рейн надел скафандр и вышел в открытый космос. Среди обломков он нашёл чёрный ящик — устройство Корпорации Омега с логотипом в виде спирали. "Саботаж. Они монополизируют квантовые ядра, чтобы поднять цены на энергию по всей Империи".

Вернувшись на борт, Рейн связался с базой: "Адмирал, это Омега. У них есть агент внутри Флота". Но связь прервалась помехами. Эхо вспыхнула тревогой: "Обнаружены три корвета класса 'Васп'. Они нас блокировали". Бой начался мгновенно. Плазменные залпы осветили вакуум, 'Нова-7' маневрировала между астероидами, уклоняясь от торпед. Рейн лично управлял турелью, сбивая врагов один за другим. Последний корвет взорвался в ослепительной вспышке.

Но победа была пирровой. Гипердвигатель повреждён, и 'Нова-7' дрейфовала к ближайшей планете — Тenebris-4, миру-улью с 20 миллиардами жителей, где правят кланы контрабандистов. Приземлившись в доках Нижнего Уровня, Рейн встретил местного информатора — эльфийку по имени Лира, с глазами цвета изумрудов и татуировками нейронных цепей. "Ты ищешь Омегу? Они здесь, в Подземелье. Но цена — 5000 кредитов и услуга", — шепнула она.

Рейн согласился. Лира провела его через лабиринт туннелей, где гудели фабрики дронов и торговали органами. В сердце Подземелья они нашли лабораторию: ряды криокапсул с квантовыми ядрами, охраняемые киборгами. Битва вспыхнула снова — лазеры, виброклинки, хакерские атаки на импланты. Лира отключила щиты, Рейн забрал ядра и поджёг лабораторию. Взрыв сотряс улей.

Улетая с Тenebris-4, Рейн и Лира стали партнёрами. "Куда дальше?" — спросила она. "В сердце Империи. Омега заплатит за всё". Их путь лежал через Пустоши Варпа, где обитали древние ужасы — тени забытых цивилизаций. Но это была лишь начало...
